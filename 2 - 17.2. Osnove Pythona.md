Prišli smo do problema, da bi morali to ponoviti za vsak trikotnik. Kaj je odgovor? Naredimo funkcijo, da to zadevo avtomatiziramo.

```python
def pl_trik(a, b, c):
	s = (a + b + c) / 2
	return (s * (s-a) * (s-b) * (s-c)) ** 1/2
def povrsina_tetraedra(x,y,z,u,v,w):
	trik1 = pl_trik(x, y, z)
	trik2 = pl_trik(x, u, w)
	trik3 = pl_trik(u, v, z)
	trik4 = pl_trik(v, w, y)
	return trik1 + trik2 + trik3 + trik4
```

V funkciji lahko tudi nekje vmes napišemo return. Ko izvede return, gre ven iz funkcije.

Pomembno je ločiti razliko med return in print:

```python
def prva(x):
	return x
	
def druga(y):
	print(y)
```

Ker vsaka funkcija nekaj vrne, če ne damo return stavka, vrne None oziroma 0 (v smislu praznine, ne številke).
Nov tip spremenljivke: <span style="color:rgb(0, 176, 80)">None</span> - to je ničla, praznina, ničesar ni

Lahko tudi iz principa to dodamo notri:

```python
def tretja(y):
	print(y)
	return None
```

Tretja funkcija je ekvivalentna drugi.

Če slučajno damo dva return stavka enega za drugim (brez kakih pogojev), ni nič hudega, samo se bo samo prvi return izvedel.
