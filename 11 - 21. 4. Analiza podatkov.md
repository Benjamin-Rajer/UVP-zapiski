Spoznali bomo <span style="color:rgb(0, 176, 80)">regularne</span> <span style="color:rgb(0, 176, 80)">izraze</span> (regex), ki se jih uporablja za iskanje po datotekah. Orodje je zelo fleksibilno, z njim lahko veliko delamo.

V Pythonu je to standardna knjižnica:

```python
import re

re.search("hello", "hello, world!") # pove nam, kje se prvič nahaja tak niz
re.finditer("bla","blablabla") # vrne nam nek seznam z vsemi pojavitvami

# lahko tudi gremo čez ta seznam:
for j in re.finditer("bla","blablabla"):
	print(j)

```

___
Opomba:
1. knjižnica re išče samo neprekrivajoče se vzorce. Ko enega najde, potem naslednjega išče tam, kjer je prejšnjega končal, ne morejo se najdene zadeve prekrivati.

```python
re.finditer("bb","bbbbbbbb") # bo najprej prva dva b našel, potem naslednja dva, ne bo pa dal skupaj drugi in tretji b
```

2. Program dela požrešno - proba vključiti čim več teksta - lahko pa ga spremenimo, da postane minimalen - da proba najti čim manjšo zadevo
3. Regularni izrazi se pojavljajo tudi drugod, npr. ko iščemo znotraj VSCode-a z uporabo ctrl+f. 
___
#### Znaki in pravila
1.  . - pika - je katerikoli znak
2. \d je števka - pazi, ker Python nekatere \ tretira kot posebne znake - v resnici bi morali uporabiti \\\d - je pa še ena rešitev - pred niz damo črko r kot raw:
```python
re.finditer(r"\d\d\d", "Imamo 500g moke.") # tale r je zelo koristen
```
 3. \w so črke in števke
 4. \s je za presledke
 5. Če za prejšnje tri damo veliko začetnico, bo nam našel ravno obratno od tega
 6. Imamo še zavite in oglate oklepaje za dodatno bolj natančno iskanje
 7. plus znak - npr. \d+ bo vzelo eno ali več tega, kar iščemo - program dela požrešno in proba dobiti čim več 
 8.  zvezdica - nič ali več pojavitev
 9. za točno določeno število damo v zavite oklepaje, npr. {3} ali pa med 1 in 5 bi bilo {1, 5}
 10. ? - vprašaj - za + ali * - damo ga, da uničimo požrešnost in da proba program najti čim manj
 Drugače pa ? pomeni 0 ali 1 znak - lahko je znak, lahko pa ga ni

Ukazov je še ogromno, obstaja tudi veliko različnih regularnih izrazov - mi moramo poznati samo te osnovne.

Gremo nazaj na naš cilj: da izluščimo podatke iz spletnih strani

V HTML datoteki iščemo nek vzorec - uporabimo regularne izraze, da to naredimo.

Odprimo datoteko in poiščimo:

```python
import re

dat = open("filmi_top_250.html")
vsebina = dat.read()
dat.close()
filmi = []
for najdba in re.finditer(r'<a\s+href="https://www\.imdb\.com/title'
r'tt (?P<id>\d+).+?text">\s*(?P<naslov>.+?)\s*</h3></a>'
r'.*?item"\s+>\s+(?P<leto>\d+)\s+</li>,
vsebina, 
flags=re.DOTALL): 
	filmi.append({"naslov": najdba["naslov"], "id": najdba["id"]}
	
print(filmi) # dobimo 250 najdb, načeloma
```
___
Opomba:
1. ko iščemo, moramo paziti s kakimi znaki, npr. pike (ker je tudi pseben znak) - če iščemo prav pike, damo backslash pred njo 
2. pika pomeni katerikoli znak, razen nova vrstica - zato moramo dodato še flags=.... da to popravimo.
3. moramo paziti, ko iščemo z regex, ker se lahko hitro zafkrnemo in kaj izgubimo - moramo biti natančni
___
Nas zdaj ne zanima ravno cela najdba, zanima nas, kar je notri.

Znotraj regularnega izraza lahko označimo neko skupino, ki označuje tisto, kar nas zanima - to je tudi uporabljeno gor.

```python

	# skupine lahko tudi poljubno poimenujemo - uporabljeno zgoraj

```

Imenuje se pa tako ker je HTML kot neka "juha" vseh teh podatkov. 

```python
from bs4 import BeautifulSoup
import json

dat = open("filmi_top_250.html")
vsebina = dat.read()
dat.close()

# zdaj pa ne uporabljamo regularnih izrazov, ampak zadeve iz beautiful soup

juha = Beautifulsoup(vsebina, "html.parser") # parse pomeni razbiti na manjše delčke
filmi = []
for najdba in juha.find_all("li", attrs={"class": "ipc-metadata-..."}):
	link = najdba.find("h3")
	urejen_link = (link.text.strip()) # našči smo naslove filmov
	
	leto = najdba.find("li", attrs={"class":"ipc-inline-..."})
	
	leto = leto.text.strip()
	najdba1 = re.search(r"\d{4}", leto)
	urejeno_leto = int(najdba1.string)
	
	filmi.append({"naslov":link.text.strip(), "leto":urejeno_leto})
	
dat = open("top250.json","w")
json.dump(filmi, dat)
dat.close()
```

Uporabili smo še attrs, ki dodatno pove neke lastnosti tega, kar iščemo - npr. kakšen class je.
